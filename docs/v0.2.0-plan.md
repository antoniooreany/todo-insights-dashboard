# v0.2.0 Analytics Plan

## Goal

Expand Todo Insights Dashboard from a basic analytics demo into a more portfolio-ready project with:
- clearer task lifecycle fields,
- stronger SQL and Pandas analysis,
- reusable KPI definitions,
- more explicit visual storytelling in charts and README.

## Release objective

Version `v0.2.0` should answer five core analytics questions:

1. How many tasks are created by day or week?
2. What is the overall completion rate?
3. How many tasks are overdue?
4. What is the average time to completion?
5. Which categories most often remain open?

The release should make the repository look more like a small analytics case study than only a coding exercise.

## Scope

This version will include:
- improved todo schema with lifecycle dates,
- reproducible demo dataset,
- SQL queries for core KPI calculations,
- Pandas-based summary generation,
- 3 portfolio-friendly charts,
- updated README sections for metrics and insights.

This version will not include:
- authentication,
- multi-user support,
- live web dashboard filtering,
- production deployment changes.

## Data model

### Proposed `todos` schema

```sql
CREATE TABLE todos (
    id INTEGER PRIMARY KEY,
    title TEXT NOT NULL,
    description TEXT,
    category TEXT NOT NULL,
    status TEXT NOT NULL CHECK (status IN ('open', 'in_progress', 'completed')),
    priority TEXT CHECK (priority IN ('low', 'medium', 'high')),
    created_at TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP,
    due_date TEXT,
    completed_at TEXT
);
```

## KPI definitions

### 1. Tasks created over time
Measure incoming workload by grouping tasks on `created_at`.

### 2. Completion rate

```text
completion_rate = completed_tasks / total_tasks * 100
```

### 3. Overdue tasks

A task is overdue when:
- `due_date` is earlier than today,
- and `status != 'completed'`.

### 4. Average completion time

```text
completion_time = completed_at - created_at
average_completion_time = mean(completion_time)
```

### 5. Open tasks by category
Group tasks where `status != 'completed'` by `category`.

## SQL queries

### A. Tasks created by day

```sql
SELECT
    date(created_at) AS created_day,
    COUNT(*) AS tasks_created
FROM todos
GROUP BY date(created_at)
ORDER BY created_day;
```

### B. Overall completion rate

```sql
SELECT
    COUNT(*) AS total_tasks,
    SUM(CASE WHEN status = 'completed' THEN 1 ELSE 0 END) AS completed_tasks,
    ROUND(
        100.0 * SUM(CASE WHEN status = 'completed' THEN 1 ELSE 0 END) / COUNT(*),
        2
    ) AS completion_rate_pct
FROM todos;
```

### C. Overdue task count

```sql
SELECT
    COUNT(*) AS overdue_tasks
FROM todos
WHERE due_date IS NOT NULL
  AND date(due_date) < date('now')
  AND status != 'completed';
```

### D. Average completion time in days

```sql
SELECT
    ROUND(AVG(julianday(completed_at) - julianday(created_at)), 2) AS avg_completion_days
FROM todos
WHERE status = 'completed'
  AND completed_at IS NOT NULL;
```

### E. Open tasks by category

```sql
SELECT
    category,
    COUNT(*) AS open_tasks
FROM todos
WHERE status != 'completed'
GROUP BY category
ORDER BY open_tasks DESC, category;
```
